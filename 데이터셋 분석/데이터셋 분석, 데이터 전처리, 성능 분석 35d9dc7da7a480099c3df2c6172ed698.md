# 데이터셋 분석, 데이터 전처리, 성능 분석

담당자: 오준혁
마지막 수정일: 2026년 5월 11일
상태: 최신
유형: 📊 분석 · 전략

**1. 개요 (Overview)**
본 프로젝트의 핵심인 인공지능(NLP, ML) 모델이 학습할 수 있도록, 인터넷상에 산재한 **비정형 데이터**를 수집하여 하나의 **표준화된 마스터 데이터셋**으로 통합하는 공정을 수행함. 데이터는 모델의 '연료'와 같으므로, 데이터의 질이 곧 전체 시스템의 성능을 결정한다는 전제하에 작업을 진행함.

**2. 데이터 확보 단계 (Data Acquisition)**
기획서에 명시된 모든 데이터 소스를 단 하나도 누락하지 않고 수집하여 **데이터의 다양성과 볼륨**을 동시에 확보함.

**📧 2.1 이메일 텍스트 데이터 (NLP용)**
• **Enron & SpamAssassin:** 실제 기업 이메일 및 연구용 스팸 코퍼스 (정상/스팸의 표준)
• **Kaggle Phishing Email:** 최신 피싱 메일 패턴 대량 포함
• **Nazario & CEAS_08:** 기획서 8.1에서 명시한 핵심 피싱 데이터셋 전량 확보
• **Ling & Nigerian_Fraud:** 소수 특수 스팸 유형까지 포괄하여 탐지 범위 확장
**🔗 2.2 악성 URL 데이터 (ML용)**
• **malicious_phish:** PhishTank 및 Kaggle 기반의 대용량(65만 건) 악성 URL 데이터
• **PhiUSIIL (UCI/Research):** 최신 보안 특징값이 포함된 고품질 전문 데이터셋 (23만 건)

**3. 세부 작업 공정 (Data Engineering Pipeline)**
실제 처리된 로직을 단계별로 기술함.

**✅ Step 1: 데이터 스키마 표준화 (Schema Alignment)**
각기 다른 파일들의 이름표(Header)를 분석하여 하나의 시스템으로 통합함.
• **입력값 통일:** `body`, `text_combined`, `Message` 등 제각각인 컬럼명을 `text`(이메일용)와 `url`(URL용)로 일치시킴.
• **라벨 규격화:** `0 = 정상(Normal)`, `1 = 피싱(Phishing)`으로 모든 데이터셋의 정답지를 통일함.

**✅ Step 2: 텍스트 노이즈 세척 (Text Cleaning)**
기획서 8.3에 근거하여 NLP 모델이 글의 '문맥'에만 집중할 수 있게 방해 요소를 제거함.
• **HTML 태그 제거:** `<br>`, `<div>` 등 기계적인 코드를 제거하여 순수 텍스트만 추출.
• **URL 마스킹 처리:** 본문 내 개별 URL을 `[URL]`이라는 공통 토큰으로 치환. (특정 URL에 모델이 과하게 학습되는 '과적합' 방지)
• **특수문자 필터링:** 한글, 영문, 숫자, 주요 문장부호 외의 불필요한 기호를 제거하여 데이터 가독성 확보.

**✅ Step 3: 데이터 무결성 검증 및 라벨 반전 교정 (Label Reversal)**
• **문제점 발견:** `PhiUSIIL` 데이터셋은 원본이 `0=피싱, 1=정상`으로 우리 프로젝트 표준과 반대로 설계되어 있었음.
• **해결책:** 수동 데이터 전수 조사를 통해 이를 확인하고, **라벨 비트 반전 연산**을 통해 프로젝트 표준(`1=피싱`)으로 100% 교정 완료함.

**4. 최종 결과물 (Final Output)**
모든 공정을 거쳐 생성된 최종 마스터 파일의 제원은 다음과 같음.**최종 파일명데이터 행 개수기획서 준수 항목담당 팀원`NLP_MASTER_DATA.csv`약 11만 건**8.1 이메일 데이터셋 계획**1번 (NLP)`URL_MASTER_DATA.csv`약 89만 건**8.2 URL 데이터셋 계획**2번 (URL)**

**5. 팀원별 액션 가이드 (To-Do List)**
 다음 팀원들은 아래 지침에 따라 모델링을 진행해주십시오.

**🙋‍♂️ 1번 팀원 (NLP 모델러)**
• **데이터 활용:** `cleaned_text` 컬럼을 입력값으로 사용하여 BERT 모델 파인튜닝을 진행하십시오.
• **주의사항:** 본 데이터셋은 이미 1차 전처리가 끝난 상태이므로, 추가적인 정제 없이 토크나이징 단계로 바로 넘어가셔도 무방합니다.

**🙋‍♀️ 2번 팀원 (URL 탐지 모델러)**
• **데이터 활용:** `url` 리스트를 기반으로 피처 엔지니어링(URL 길이, 도메인 특성 추출 등)을 수행하여 XGBoost 학습을 진행하십시오.
• **주의사항:** 89만 건의 대용량 데이터이므로, 학습 시 메모리 관리에 유의하며 교차 검증을 수행하십시오.

**👤 3번 (본인) 향후 계획**
• 팀원 1, 2가 뽑아온 모델별 '예측 확률값'을 전달받아 **앙상블 엔진**에 태워 최종 정확도를 분석할 예정입니다.
• 시스템 전체의 오탐률(FPR)을 분석하여 실무 적용 가능성을 검증하는 차트 및 성능 보고서를 제작할 것입니다.
